    Create a new FastAPI router with prefix `/api/system-mappings`:

    - `GET /api/system-mappings?document_id={uuid}` — List all system mappings for a document. Join through ActionItem -> Obligation to filter by document_id.     - `PATCH /api/system-mappings/{mapping_id}/review` — Confirm, correct, or reassign a mapping. Use this Pydantic request model:
      ```python
      class EngineeringReviewRequest(BaseModel):
          action: Literal["confirmed", "corrected", "reassigned"]
          actor: str = "engineer"
          corrected_system: str | None = None  # Required for corrected/reassigned
          reason: str | None = None  # Required for corrected/reassigned
      ```
      Validation: if action is "corrected" or "reassigned", require corrected_system and reason (raise 422 if missing).
      Use the EXACT same atomic dual-write pattern from obligations.py:
      - `async with async_session_maker() as session:` then `async with session.begin():`
      - Fetch mapping, 404 if not found, 409 if already confirmed
      - Set `mapping.confirmed = True` and `mapping.reviewed_at = func.now()`
      - If corrected/reassigned: save old `system_name`, update `mapping.system_name = body.corrected_system`, set `mapping.engineer_note = body.reason`, set `mapping.suggested_by = "engineer"`
      - Insert AuditLog with entity_type="system_mapping", entity_id=mapping_id, action=body.action, actor=body.actor, metadata_={"previous_system": old_system} for corrections or None for confirmations
      - After transaction commits, emit Kafka event: `KafkaTopic.SYSTEM_MAPPING_CONFIRMED` with `{"mapping_id": str(mapping_id), "action_item_id": str(mapping.action_item_id)}`
      - Return the updated mapping as a dict
