**1. Create backend/ride/workers/parse_worker.py** -- the PDF-to-Markdown parse worker:
- `class ParseWorker(BaseConsumer):` subclassing the existing BaseConsumer ABC
- `__init__`: call `super().__init__(KafkaTopic.DOCUMENT_UPLOADED, group_id="parse-worker")`
- Implement `async def process(self, message: dict) -> None:`
  - Extract `doc_id = message["document_id"]` and `file_path = message["file_path"]`
  - Call `pymupdf4llm.to_markdown(file_path)` to convert PDF to structure-preserving Markdown. NOTE: `pymupdf4llm.to_markdown()` is a synchronous call. Wrap in `asyncio.to_thread()` to avoid blocking the event loop: `md_text = await asyncio.to_thread(pymupdf4llm.to_markdown, file_path)`
  - Open an async session via `async with async_session_maker() as session:`, fetch the Document by ID using `select(Document).where(Document.id == uuid.UUID(doc_id))`, update `doc.content_markdown = md_text` and `doc.status = "parsed"`, then `await session.commit()`
  - After DB update, emit to next pipeline stage: create a temporary BaseProducer, start it, send `KafkaTopic.DOCUMENT_PARSED` with payload `{"document_id": doc_id}`, stop it. Alternatively, hold a producer instance on self to avoid repeated start/stop -- add `self._producer = BaseProducer()` in __init__, start it in a helper or expect it to be started externally. Simplest approach: create and manage a `BaseProducer` as `self._emit_producer` initialized in __init__, started in an overridden `start()` method (call `await super().start()` first, then `await self._emit_producer.start()`), and stopped in an overridden `stop()` method.
