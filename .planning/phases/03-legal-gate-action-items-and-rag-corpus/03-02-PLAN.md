**H. Backend: Add GET /api/documents endpoint:**

Add to `backend/ride/api/routes/documents.py` a new GET endpoint:
```python
@router.get("/")
async def list_documents(db: AsyncSession = Depends(get_db)) -> list[dict]:
    result = await db.execute(
        select(Document).order_by(Document.uploaded_at.desc())
    )
    documents = result.scalars().all()
    return [
        {
            "id": str(d.id),
            "filename": d.filename,
            "status": d.status,
            "content_markdown": d.content_markdown,
            "uploaded_at": d.uploaded_at.isoformat() if d.uploaded_at else None,
        }
        for d in documents
    ]
```

Also add a GET single document endpoint:
```python
@router.get("/{document_id}")
async def get_document(document_id: uuid.UUID, db: AsyncSession = Depends(get_db)) -> dict:
    result = await db.execute(select(Document).where(Document.id == document_id))
    doc = result.scalar_one_or_none()
    if not doc:
        raise HTTPException(status_code=404, detail="Document not found")
    return {
        "id": str(doc.id), "filename": doc.filename, "status": doc.status,
        "content_markdown": doc.content_markdown,
        "uploaded_at": doc.uploaded_at.isoformat() if doc.uploaded_at else None,
    }
```

**D. ObligationCard** (`frontend/src/components/review/ObligationCard.tsx`):

This is the most important component — each card represents one obligation the reviewer acts on. Per user decision:

1. **Card structure** using shadcn Card component:
   - Header: Obligation number/title + ambiguity badge (if `is_ambiguous`)
   - AI Summary: The `text` field displayed prominently
   - Source Quote: The `source_quote` field displayed in a visually distinct block — use a left-bordered blockquote style with a slightly different background (e.g., slate-50 bg, left border in teal/primary). This is the evidence the reviewer reads.
   - Reasoning: The `reasoning` field in a collapsible or always-visible section with a subtle "AI Reasoning" label — shows chain-of-thought
   - Action buttons: Large, prominent Approve (green/teal) and Reject (red/destructive) buttons

2. **Ambiguity badge**: If `is_ambiguous === true`, show a Badge component with "Ambiguous" text in an amber/warning color. Per user decision: `is_ambiguous` obligations must be visually distinct so reviewer pays extra attention.

3. **Status states**:
   - `pending`: Show Approve/Reject buttons, card is interactive
   - `approved`: Show green "Approved" badge, no action buttons, card is muted/dimmed slightly
   - `rejected`: Show red "Rejected" badge, no action buttons, card is muted/dimmed slightly

4. **Review handler**: When Approve or Reject is clicked:
   - Set loading state on the button
   - Call `reviewObligation(obligation.id, action)` from api.ts
   - On success, call `onReview(obligation.id, action)` callback to update parent state
   - On error (409 = already reviewed), show the current status
   - On other errors, show error message

5. **Design per user decisions**:
   - Large, prominent action buttons (not tiny icons)
   - Clean card with subtle shadow
   - Generous whitespace between sections
   - Source quote visually highlighted — must be readable side-by-side with left panel document
   - One-at-a-time review experience — each card is self-contained

**Color palette specifics:**
- Approve button: Teal/green variant — use shadcn Button with custom class `bg-emerald-600 hover:bg-emerald-700 text-white`
- Reject button: Red/destructive variant — use shadcn Button `variant="destructive"`
- Ambiguous badge: Amber/yellow — `bg-amber-100 text-amber-800 border-amber-200`
- Approved badge: Green — `bg-emerald-100 text-emerald-800`
- Rejected badge: Red — `bg-red-100 text-red-800`
- Source quote block: `bg-slate-50 border-l-4 border-primary/30 pl-4 py-3`

**Important UX detail:** The obligation cards on the right should scroll independently of the document on the left. The reviewer reads the document on the left and reviews obligations on the right — both panels scroll independently via ScrollArea.
